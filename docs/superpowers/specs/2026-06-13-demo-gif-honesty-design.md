## Проблема

Demo-гифка проекта выдаёт за живой прогон то, что на самом деле напечатано, и
существует в трёх рассинхронизированных копиях.

**Факты (проверены через Read/grep, не по памяти):**

1. **Сцена живёт в 3 местах:**
   | Артефакт | Что это | `/etc/hosts` строка | `config_correlate` реален? | Упомянут |
   |---|---|---|---|---|
   | `assets/demo.gif` | offline-рендер `make_gif.py`, всё нарисовано | `:12` ❌ | нет | **README:16 `<img>` — это видит зритель** |
   | `assets/demo.svg` | offline SVG, всё нарисовано | `:12` ❌ | нет | fallback, в README напрямую не вставлен |
   | `assets/cli.gif` | VHS-запись `cli.mjs`, `config_correlate` живой | `:2` ✅ | да (1 из 3) | README:214–218 — **файла нет в репо** |

2. **README показывает наименее честный артефакт.** `<img src="assets/demo.gif">`
   (README:16) — 0/3 живых вызовов. Честный-на-⅓ `cli.gif` лишь упомянут текстом и
   в репозитории отсутствует (генерируется только Action `regenerate demo gif`).

3. **`:12` против `:2`.** `make_gif.py` (ROW3) и `demo.svg` пишут `/etc/hosts:12`, но
   `demo/hosts.fixture` кладёт `api.acme.dev` на **строку 2**. `cli.gif` показывает
   правду (`:2`), т.к. читает живой ответ `config_correlate`.

4. **Подпись приписывает заслугу не тому инструменту.** Все три рендера: «one
   net_diagnose call». Настоящий money-shot — `config_correlate`, а не `net_diagnose`.

5. **Пролог в `cli.mjs` захардкожен не по лени — это обход реальных ограничений:**
   - `net_diagnose` к `api.acme.dev` в офлайне: несуществующий домен → `resolveDns`
     пуст → первая ветка `tools.ts:351` возвращает «DNS resolution FAILED», красивый
     `TCP ✗ timeout @10.0.0.5` недостижим.
   - `net_triangulate` в офлайне: `guard.assertNetworkAllowed("globalping")` при
     `localOnly` бросает (`guard.ts:101`) — инструмент не отрабатывает вообще.
   - `config_correlate` в офлайне: читает фикстуру, для `api.acme.dev` живой
     `resolveDns` → нет ответа → ветка `!liveIp` (`tools.ts`): «/etc/hosts:2 pins
     api.acme.dev → 10.0.0.5; this OVERRIDES DNS». **Работает вживую, честно, `:2`.**

**Вывод:** «сделать пролог живым выводом» в офлайне невозможно (несуществующий домен +
отрезанный Globalping). Цель — не подделать живой вывод, а **перестать им
притворяться**, оставив единственный реально живой вызов (`config_correlate`) как
кульминацию.

## Решения (приняты пользователем)

| Развилка | Решение |
|---|---|
| Уровень честности | **Честность по форме** — пролог остаётся напечатанным, но не маскируется под живой вывод. |
| Канонический артефакт | **`cli.gif` (VHS)** — единственный. `make_gif.py` + `demo.gif` + `demo.svg` удаляются. |
| Как пометить пролог | **Вариант B** — пролог подан как *план диагностики агента* (намерение), не как полученный результат. |
| Способ рендера `cli.gif` | **CI `demo.yml`** (workflow_dispatch, рендерит и коммитит). Машину пользователя не трогаем. |
| Порядок | **Подход 1 — два коммита**, README ни секунды не битый. |

## Дизайн по компонентам

### 1. `demo/cli.mjs` — money-shot как план (вариант B)

Блок строк 45–66 («the money shot») переписывается. Механика MCP-клиента
(строки 22–43), реальный RPC `config_correlate` (строка 57) и `srv.kill()` —
**не меняются**.

**Было (печатает выдуманный результат):**
```
net_diagnose      DNS ✓  TCP ✗ timeout @10.0.0.5      ← строка 52, фейк-результат
net_triangulate   you ✗   US ✓  EU ✓  Asia ✓          ← строка 54, фейк-результат
config_correlate  /etc/hosts:2  api.acme.dev → 10.0.0.5  ← строка 59, live
┃ — one diagnosis · local-first · zero telemetry      ← строка 66, заслуга не тому
```

**Станет (печатает план + live-кульминацию):**
```
❯ why can't I reach api.acme.dev?

# an agent probes top-down — DNS, TCP, then worldwide:
net_diagnose      → DNS · ping · TCP · TLS · HTTP        (cyan tool + dim arrow-text)
net_triangulate   → here vs US / EU / Asia               (cyan tool + dim arrow-text)

# but the catch no remote probe can make:
config_correlate  /etc/hosts:2  api.acme.dev → 10.0.0.5   ● live   ← реальный RPC
┃ ❯ It's your side.
┃ /etc/hosts:2 pins api.acme.dev -> 10.0.0.5; this OVERRIDES DNS itself   ← живой
┃ returns nothing). If api.acme.dev seems stuck on an old address, this     finding,
┃ line is why.                                                              wrap, 3 стр
┃ It's live from US, EU & Asia — remove that line.
┃ — config_correlate is a real call · local-first · zero telemetry
```

> **ВАЖНО (находка ревью, уточнено замером):** живой `finding` = 157 символов. Текущий
> `cli.mjs:64` режет его `slice(0, 78)` → обрывает на полуслове (`…DNS itself ret`) —
> оборванное предложение попало бы в gif. Вариант B **обязан** убрать `slice` и сделать
> word-wrap живого `finding` по словам. **Замер на реальном тексте: при ширине ~78
> симв/строка это 3 строки, НЕ 2** (157 симв не помещаются в 2×76 красиво — на 72
> отрывается `(DNS`, на 80 одиноко висит `why.`). Принято: ширина ≈78, **3 строки**,
> живой текст целиком. Своя «причёсанная» строка вместо живого вывода — ЗАПРЕЩЕНА (это
> вернёт фейк туда, где B обещает единственный настоящий вызов). Текст вердикта — РОВНО
> то, что вернёт `config_correlate` в рантайме. Точный код wrap-функции — в плане.

**Точные изменения:**
1. Строки `net_diagnose`/`net_triangulate` показывают **что инструмент проверяет**
   (`→ DNS · ping · TCP · TLS · HTTP` / `→ here vs US / EU / Asia`), а не выдуманный
   результат. Это правда: `tools.ts:332` описывает ровно эти шаги.
2. Два комментария-ремарки `#` (`an agent probes top-down…` / `the catch no remote
   probe can make:`) создают рамку «план → но вот единственное, что remote-зонд не
   увидит». Граница «намерение / живой вызов» становится явной.
3. `config_correlate` помечается зелёным маркером `● live` — единственная строка,
   обозначенная как настоящий вызов.
4. Подпись: `config_correlate is a real call` (вместо `one diagnosis`).
5. `/etc/hosts:2` берётся из живого `finding` (как сейчас) — правда гарантирована
   кодом, не хардкодом. Хардкод-строка с `:2` (текущая строка 59) остаётся синхронной.
6. **Вердикт: снять `slice(0, 78)` (строка 64), word-wrap живого `finding`.** Реальный
   `finding` = 157 симв, текущая обрезка рвёт слово (`ret…`). Перенос по словам, ширина
   ≈78 → **3 строки** (замерено, см. блок ВАЖНО). Находка ревью.

**Точный текст комментариев `#` одобрен пользователем** (это маркетинговый посыл,
который зритель прочтёт в кадре).

## Тестирование и верификация

**Уровень 1 — Claude верифицирует перед коммитом A (без vhs):**
1. `npm run build` зелёный.
2. `node demo/cli.mjs` напрямую (НЕ требует vhs — обычный Node + MCP-клиент к
   `dist/index.js`). На macOS НЕ оборачивать в `timeout` (его нет в BSD; cli.mjs сам
   делает `srv.kill()` + `process.exit(0)`). Проверить в выводе:
   - `config_correlate` реально вызвался, `finding` содержит `/etc/hosts:2`;
   - **вердикт показан целиком, БЕЗ обрыва на полуслове** (нет `…ret`, есть полное

     не «причёсан» вручную;

   - подпись = `config_correlate is a real call`.
   Это **главная проверка честности** — `cli.mjs` исполняется как есть.
3. `npm run smoke` — MCP-handshake не сломан (точное число инструментов не
   фиксируем: `tools.ts` не трогается, smoke сам ассертит актуальное значение).

## Критерий приёмки

Зритель гифки понимает, что:
1. `config_correlate` — настоящий вызов;
2. две строки над ним — что агент *запустил бы*, а не сфабрикованный вывод;
3. ни один remote-зонд не поймал бы stale-pin.

И README ни в один момент не битый.

## Заметка по процессу

При кликах в визуальном компаньоне пользователь сначала рассмотрел вариант C («только
реальный вызов, 100% правды»), затем выбрал B. Интерес к C означает: при записи
критично, чтобы граница «план / живой вызов» в B читалась **с первого взгляда** —
иначе соблазн «C был честнее» вернётся. Это учтено в критерии приёмки (п. 2) и
чек-листе Уровня 2.
